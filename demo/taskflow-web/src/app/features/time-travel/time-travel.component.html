Time travel allows you to reconstruct the state of any aggregate at any version by replaying events up to that point.

Key Event Sourcing Concepts:

  • Event Replay - Reconstruct past state by replaying stored events
  • Audit Trail - Complete history of all changes and who made them
  • Temporal Queries - Answer questions about the past ("What was the project status on January 15th?")
  • Event Versioning - Navigate through aggregate versions as events were applied
  • State Snapshots - View the exact state at any event boundary

Features:

  • View project state at any historical version
  • See work item changes over time
  • Inspect team composition at different points
  • Understand how aggregates evolved
  • Debug issues by replaying event sequences
